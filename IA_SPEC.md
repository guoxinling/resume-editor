## 一、产品定位

**一句话**：让 AI 写出面试官想看的简历。

**差异化**：
- 预览居中，所见即所得
- 对话式 AI 统一入口，不用在多个 Tab 间切换
- 本地优先 + 隐私保护（无需注册即可试用）
- 面试预测 + AI 答案建议（竞品缺失）

**目标用户**：
- 2-8 年经验的职场人（非应届生主战场）
- 海外求职者（中英双语）
- 隐私敏感者（本地存储优先）

## 三、页面详细设计

### 3.1 Landing Page

**交互规则**：
- 顶部 Tab "AI写简历"（默认选中）/ "模版市场" 切换
- 点击"模版市场" → 内容区变为模板卡片网格，Hero 区缩小或消失
- 点击"开始对话" → Wizard 启动（无需登录）
- 点击"导入简历" → 文件选择器（无需登录）
- 点击"注册/登录" → Magic Link 弹窗

**底部 CTA**：
- 紫色面板（primary-container），左右分栏
- 左侧：大标题 "准备好开始整理你的下一份简历了吗？" + 描述文字
- 右侧：竖排两个按钮 — "AI 帮我从零写"（主按钮）+ "导入我的简历"（次按钮）
- 按钮带 SVG 图标，点击跳转编辑器对应模式

### 3.2 模版市场 Tab

**MVP 范围**：8 套模板，每套 = 预设 CSS + 模块排列顺序，不改数据结构。

模板分类（待定）：应届生 / 职场进阶 / 技术产品 / 海外双语 / 金融精英 / 创意作品 / 时间轴 / 极简白纸

### 3.3 Dashboard（简历管理中心）

**注意**：未登录用户跳过 Dashboard，直接进入编辑器（本地模式）。

### 3.4 Editor（核心工作区）

**核心交互规则**：

| 操作 | 行为 |
|------|------|
| 点击预览区模块 | 右侧面板切换为编辑表单（该模块） |
| 点击文本旁的 ✨ | 右侧面板切换为 AI 模式，自动填入润色指令 |
| 点击 💬 AI助手 | 右侧面板展开，AI 对话模式 |
| 点击右侧面板 ✕ | 恢复默认状态（无面板） |
| 拖拽导航栏模块 | DnD 重排序，预览实时同步 |
| 点击导出 | 免费用户加水印，引导注册/付费 |

**Header 精简方案**：

| 旧（12项） | 新（4元素 + 1 CTA） | 去向 |
|-----------|-------------------|------|
| 导入 | — | 移到 Dashboard + Landing |
| 保存 | 自动保存（已有） | IndexedDB 2s 自动保存 |
| 草稿 | — | 移到 Dashboard |
| 导出 PDF | ⬇ 导出 ▾ | 合并为导出下拉 |
| 导出 PNG | ⬇ 导出 ▾ | 合并为导出下拉 |
| 语言 | CN/EN | 保留，精简文字 |
| 新建 | — | 移到 Dashboard |
| AI 工具 | 💬 AI | 保留，改为开关按钮 |

## 四、AI 交互方案：对话 + 快捷胶囊（混合模式）

### 4.1 设计原则

- **默认对话**：打开 AI 面板即进入对话，AI 主动引导
- **快捷胶囊**：对话顶部常驻 4 个胶囊，一键触发对应能力
- **底层复用**：对话背后仍是现有的 4 组 AI 能力，只是 UI 统一

### 4.2 面板布局

```
┌────────────────────────────────────┐
│ 💬 AI 助手                    [✕] │
├────────────────────────────────────┤
│                                    │
│ [✨ 润色经历] [🎯 JD匹配]         │
│ [🔍 诊断简历] [💬 面试问题]       │
│                                    │
│ ─────────────────────────────      │
│                                    │
│ 🤖 AI：你好！我可以帮你优化简历。  │
│ 你可以直接告诉我：                │
│ · 想润色哪段经历                  │
│ · 粘贴目标JD，分析匹配度          │
│ · 整体诊断你的简历                │
│ · 生成面试问题                    │
│                                    │
│ ─────────────────────────────      │
│                                    │
│ 或者直接做这些：                  │
│                                    │
│ ┌──────────────────────────────┐   │
│ │ 💬 输入你想做的事，如"帮我看看  │   │
│ │ 第一条工作经历怎么写更好"    │   │
│ └──────────────────────────────┘   │
│                         [发送 →]   │
└────────────────────────────────────┘
```

**点击快捷胶囊的行为**：
- ✨ 润色经历 → AI 列出工作经历，问你想润色哪条、用哪种风格
- 🎯 JD匹配 → AI 让你输入 JD（可粘贴 / OCR截图）
- 🔍 诊断简历 → AI 自动分析整份简历，输出诊断报告
- 💬 面试问题 → AI 基于当前简历生成 10 道问题 + 答案建议

## 五、商业化体系

### 5.1 点数系统

| 行为 | 消耗点数 | 说明 |
|------|---------|------|
| ✨ 经历润色 | 1 点 | 单条经历 + 风格选择 |
| 🎯 岗位适配 | 3 点 | 简历 + JD 全文分析 |
| 💬 面试预测 | 5 点 | 10 道问题 + 答案建议 |
| 🔍 简历诊断 | 5 点 | 逐模块评分 + 改写建议 |
| 📷 OCR识别 | 免费 | 纯前端 Tesseract.js |

**免费额度**：10 点/天（次日刷新）

**付费包**：

| 包名 | 价格 | 点数 | 单价 |
|------|------|------|------|
| 入门包 | ¥9.9 | 50 点 | ¥0.20/点 |
| 进阶包 | ¥29.9 | 200 点 | ¥0.15/点 |
| 求职季包 | ¥59.9 | 500 点 | ¥0.12/点 |

**成本核算**（DeepSeek V4 Pro 全价 / 折扣价）：
- 1 点 API 成本 ≈ ¥0.003-0.034（折扣）/ ¥0.014-0.168（全价）
- 毛利空间：83%-97%

### 5.2 免费 vs 付费

| 功能 | 免费版 | Pro版 |
|------|--------|-------|
| 简历编辑 | ✅ 完整 | ✅ 完整 |
| AI 调用 | 10 点/天 | 购买点数包 |
| PDF/PNG 导出 | ✅ 带水印 | ✅ 无水印 |
| 云端同步 | ❌ 仅本地 | ✅ 多端同步 |
| 多简历管理 | 1 份 | 无限 |
| 模板市场 | 基础模板 | 全部模板 |

### 5.3 水印方案

- **位置**：A4 页脚右下角
- **样式**：浅灰色 #999，12px，30% 透明度
- **内容**：「由简历鸭 AI 生成 · jianliya.app」
- **效果**：面试官能看到但不遮挡内容（类似"由微信读书排版"）

### 5.4 用户系统

- **方案**：Magic Link 邮箱登录（无密码）
- **流程**：输入邮箱 → 收验证链接 → 自动登录
- **技术栈**：Supabase Auth + Resend SMTP（发邮件）+ Supabase PostgreSQL（数据存储）
- **支付**：支付宝/微信扫码 → 回调更新点数（备案后接入）

## 七、API 后端代理

### 7.1 架构

```
用户浏览器 ──→ POST /api/chat ──→ Vercel Serverless Function
                                        │
                                   ├ 校验点数余额
                                   ├ 扣减点数
                                   ├ 转发 DeepSeek API
                                   └ 返回流式响应
                                        │
                                   DeepSeek V4 Pro
```

### 7.2 用户端变化

- 删除 AI Settings Tab（用户不再需要配置 API Key）
- API Key 存储为 Vercel 环境变量
- 点数存储在 Supabase PostgreSQL

### 7.3 环境变量

```env
DEEPSEEK_API_KEY=sk-xxx
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-pro
SUPABASE_URL=...
SUPABASE_ANON_KEY=...
```

## 八、设计规范（视觉语言）

### 8.1 品牌色

```
主色渐变：indigo → purple → fuchsia
  #4F46E5 → #7C3AED → #A855F7

背景：白色为主，浅灰 #F9FAFB 为辅
文字：#1F2937（主）/ #6B7280（辅）/ #9CA3AF（弱）
边框：#E5E7EB（默认）/ #D1D5DB（hover）
```

### 8.2 圆角体系

| 元素 | 圆角 |
|------|------|
| 输入框 / 按钮 | 8px |
| 卡片 / 面板 | 12px |
| 头像 / 图标容器 | 16px |
| 全屏面板 | 0px（无边角） |

### 8.3 间距体系（4px 网格）

| Token | 值 | 用途 |
|-------|-----|------|
| xs | 4px | 图标与文字间距 |
| sm | 8px | 紧凑间距 |
| md | 12px | 默认卡片内边距 |
| lg | 16px | 段落间距 |
| xl | 24px | 模块间距 |
| 2xl | 32px | 页面分区 |
| 3xl | 48px | 大区块 |

### 8.4 字体

```
中文：系统默认（-apple-system, PingFang SC, Microsoft YaHei）
英文：Inter, system-ui
代码：JetBrains Mono, monospace

标题：font-semibold
正文：font-normal
辅助文字：text-sm / text-xs
```

### 8.5 阴影

```
卡片：shadow-sm（0 1px 2px rgba(0,0,0,0.05)）
悬浮面板：shadow-lg（0 10px 15px rgba(0,0,0,0.1)）
模态：shadow-2xl
主按钮：hover:shadow-lg shadow-indigo-500/20
```
